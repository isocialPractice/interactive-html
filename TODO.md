## Implementation Notes

### For Claude Code Automation:
1. **Follow Existing Patterns**: All tools should follow the structure of existing HTML files in the repository
2. **Single-File Architecture**: Each tool is a single HTML file with embedded CSS and JavaScript
3. **Use Template System**: Base on `/Templates/interactive.dwt` template structure
4. **Consistent Styling**: Follow the design system in `/docs/DESIGN_SYSTEM.md`
5. **Mobile Responsive**: All tools must work on mobile devices
6. **No External Dependencies**: Keep everything self-contained (no npm, no build process)
7. **Accessibility**: Include proper labels, ARIA attributes, and keyboard navigation
8. **Real-time Feedback**: Update results as users type/interact
9. **Visual Elements**: Include diagrams, charts, or visual representations where helpful
10. **GitHub Pages Compatible**: All paths must work with the GitHub Pages deployment

### Priority Levels:
- **High**: General tools "coming soon" list, additional tools for existing categories
- **Medium**: New categories aligned with site theme (math, physics, chemistry, electronics-adjacent)
- **Low**: Specialized tools for niche use cases

### Completion Criteria:
- Tool works correctly with all edge cases
- Mobile responsive design
- Consistent with existing tool styling
- Proper navigation integration
- Added to category index.html
- Tested in browser (localhost and GitHub Pages paths)
